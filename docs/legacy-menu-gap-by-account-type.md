# 레거시 메뉴 갭 인벤토리 (이미지 증거 기반)

## 기준

- 델파이 캡처 증거: `IMG_8050`, `IMG_8051`, `IMG_8052`, `IMG_8053`, `IMG_8054`, `IMG_8055`, `IMG_8056`, `IMG_8057`
- 웹 기준: `migration/contracts/rbac_menu_matrix.yaml` + `frontend/src/lib/form-registry.ts`
- 계정 기준: `T2_DIST`, `T2_PUB`, `T3`, `T3_WAREHOUSE_LITE`, `T3_WAREHOUSE_FULL`

## 레거시 확인 메뉴 (첨부 이미지)

- 출고관리: 출고접수관리, 출고접수현황, 출고택배관리, 거래명세서, 입고명세서, 반품명세서, 신간발행, 전송자료받기(FTP)
- 재고원장: 도서별수불원장, 기간별재고원장
- 반품재고관리: 재고변경(정품/반품), 반품재고(입고/폐기), 반품재고입력/현황
- 발송비/입금관리: 발송비내역, 발송비현황, 청구서관리, 입금내역, 입금현황, 출고내역서, 머지(접부장)
- 통계관리: 도서별판매, 거래처판매, 년/월 집계 계열
- 기초관리: 거래처관리(통합/개별), 출판사관리, 입고처관리, 도서관리, 특별관리, 지역분류, 환경설정

## 현재 웹 상태 갭

| 구분 | 레거시 기준 | 현재 웹 상태 | 갭 |
| --- | --- | --- | --- |
| T2_DIST | 출고/재고원장/반품/발송비·입금/내역서 모두 노출 | 사용자별로 일부 그룹 누락 사례 존재 | 메뉴 판정기 단일화 필요 |
| T2_PUB | 거래/원장/회계/자료/통계 중심 | 일부 화면에서 물류 그룹 흔적 또는 권한 혼합 | 메뉴-권한 판정 분리 필요 |
| T3 | 출판사 셸 기준 + 독립 운영 | 일부는 통계만 보이고 일부는 물류 일부 노출 | 유형 분기 단일 정책 필요 |
| T3_WAREHOUSE_LITE | 반품재고 통합형 | 통합/분리 화면이 혼재 | 서브타입 기준 명시 필요 |
| T3_WAREHOUSE_FULL | 재고/반품 분리형 | LITE 규칙과 혼합 노출 사례 | 서브타입별 메뉴 고정 필요 |

## 우선 교정 원칙

1. 메뉴 노출은 `ACC-MENU-*` 단일 판정으로만 수행 (`form-registry` / `sidebar` 통합).
2. `license_keys` 는 웹 런타임 메뉴 판정에서 제거한다.
3. 권한 UI는 F키가 아닌 한글 업무권한 중심으로 제공하고, F키는 고급 모드에만 노출한다.
4. 관리자 영역은 수퍼(T1)와 총판(T2_DIST) 화면을 분리하고 데이터 스코프를 강제한다.
