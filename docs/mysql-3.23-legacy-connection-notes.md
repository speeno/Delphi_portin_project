# MySQL 3.23 (154/155) 접속·쿼리 테스트 — 구현 메모

실 포팅·하네스·분석 도구 연동 시 참고용. **115.68.3.154 / 115.68.3.155** 의 MySQL은 **3.23.58(1999)** 로, 현대 **PyMySQL 기본 경로가 지원하지 않는 프로토콜**을 사용한다. 아래는 테스트에서 **접속 및 쿼리 실행까지 성공**하기 위해 정리한 원인·대응이다.

## 환경 요약

| 항목 | 내용 |
|------|------|
| 서버 | 115.68.3.154, 115.68.3.155 |
| MySQL 버전 | 3.23.58 |
| 현대 클라이언트 | PyMySQL 등 MySQL 4.1+ 핸드셰이크 전제 → **직접 사용 불가** |

## 문제·원인·수정

| 문제 | 원인 | 수정 |
|------|------|------|
| **2013 연결 끊김** | PyMySQL이 MySQL 4.1+ 핸드셰이크를 보냄 → 3.23이 처리 불가 | `pymysql_compat.py`에 **MySQL 3.x 전용 raw 소켓 커넥터** 구현 |
| **1045 비밀번호 거부** | `CLIENT_LONG_PASSWORD` 플래그 누락 → 서버는 scramble 대신 **raw hash** 기대 | `client_flags`에 **0x0001** OR 추가 |
| **max_allowed_packet 관련 오류** | `CLIENT_COMPRESS` 에코로 **압축 프로토콜**로 넘어가며 패킷 포맷 불일치 | **`CLIENT_COMPRESS`(0x0020) 비트 제거** |

## SSH

- **sshpass 없이** 비밀번호 SSH 인증을 위해 **SSH_ASKPASS** 메커니즘을 구현해 사용.

## 코드 위치(권장)

- 호환 레이어·raw 커넥터: 팀에서 사용 중인 **`pymysql_compat.py`**(또는 동등 모듈)를 `debug/` 또는 `tools/db/` 등 **한 곳에 고정**하고, 커밋 정책·비밀 관리 규칙에 맞게 보관한다.
- 본 문서는 **동작 요약·의사결정 기록**용이며, 실제 소스는 별도 파일로 관리하는 것을 권장한다.

## 대시보드·상태

- 호스트별 연결 요약: [`dashboard/data/db-status.json`](../dashboard/data/db-status.json)
- 재프로브(단순 경로): [`debug/db_connect_probe.py`](../debug/db_connect_probe.py) — **154/155는 기본 스크립트와 다른 클라이언트가 필요할 수 있음**

## 후속 작업 제안

- `tools/db/schema_extractor.py` 등은 **mysql-connector** 기준이므로, 3.23 서버용 스키마 덤프는 **레거시 전용 클라이언트** 또는 **서버 측 `mysqldump`(해당 버전)** 로 수집하는 방안 검토.
- 포팅 파이프라인에는 **버전별 분기**(모던 4대 vs 3.23 2대)를 명시하는 것이 안전하다.
