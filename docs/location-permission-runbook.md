# 위치 권한(브라우저) 수동 점검

`LocationPermissionBootstrap` + 내정보 화면의 저장값 초기화를 확인하는 절차. 정적 가드: `test/test_platform_region_sync_static.py`.

## 전제

- 로그인 완료, 수동 지역을 고르지 않은 상태(플랫폼 지역이 `manual` 이 아님)에서만 첫 방문 배너가 뜬다.
- `localStorage` 키 접두: `portal_location_permission_v1` (DEC-063).

## 시나리오

1. **최초/초기화 후**  
   - 앱 로드 → 우하단 「위치 정보 사용」 카드가 보이는지 확인.  
   - 「나중에」 → 배너가 사라지고, 동일 세션·재방문에서 배너가 뜨지 않는지 확인.
2. **재요청**  
   - 설정 → 내정보 → 「위치 안내 다시 요청(저장값 초기화)」 → 짧은 메시지 후 자동 새로고침.  
   - (1)과 같이 수동 지역이 아니면 배너가 다시 뜰 수 있음.
3. **허용**  
   - 「허용하고 현재 위치 사용」 → 브라우저 권한 대화 후 헤더 날씨/지역 프리셋이 갱신되는지 확인(환경·권한에 따라 상이).
4. **거부**  
   - 브라우저에서 위치 거부 → 로컬 `denied` 저장, 배너는 닫힘. 재요청 버튼으로 플래그 제거 후 다시 (1) 확인.

## 실패 시

- DevTools → Application → Local Storage 에서 `portal_location_permission_v1:` 키 제거 후 재시도.  
- 수동 지역 선택 시 배너가 의도적으로 숨겨짐 — 지역 설정을 기본으로 되돌린 뒤 반복.
