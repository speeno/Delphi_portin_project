# 교통소통정보 — 이용 코드·필드 참조 (표준노드링크 연계)

ITS OpenAPI **`trafficInfo`** 호출·결과 해석·선택 UI 구성 시 함께 쓰는 값이다. 원본은 사용자 제공 자료(포털 PDF·노선번호·노드별 링크 CSV·본선구간 엑셀)와 [its-traffic-sources.md](./its-traffic-sources.md)를 따른다.

## OpenAPI 요청 (`trafficInfo`) — 사용자 선택과 매핑

| 파라미터 | 허용 값(요약) | 비고 |
|-----------|----------------|------|
| `type` | `all`, `ex`, `its` | 전체 / 고속도로 / 국도 |
| `routeNo` | 정수 노선 ID | 포털 샘플은 `1`(경부) 등 — **노선번호 마스터의 `road_id`** 와 대응하는 경우가 많음 |
| `drcType` | `all`, `up`, `down`, `start`, `end` | 상·하행·시점·종점 등 |
| `minX`, `maxX`, `minY`, `maxY` | 경위도 실수 | `type`·`drcType`이 `all`일 때 영역 조회 |

노선 목록(고속 60건·국도 51건)은 레포에 JSON으로 두었다.

- 백엔드: `도서물류관리프로그램/backend/data/its/route_numbers.json`
- 프론트: `도서물류관리프로그램/frontend/src/lib/its-route-numbers.json` (동일 내용)

드롭다운에는 **`road_id`** 를 API `routeNo`로 보내고, 표시는 **`route_nm`**(+ 필요 시 `route_no_display`)을 쓴다.

## 도로구간(링크) 속성 — PDF 「교통소통정보_도로구간정보_설명」

소통 응답의 링크와 표준노드링크 속성을 맞출 때 사용한다.

| 개념 | API·데이터 필드명 예 | 설명 |
|------|----------------------|------|
| 링크 식별 | `linkId` / `roadsectionid` | 도로구간 ID(링크아이디). 10자리 구조: 앞 3자리 권역 등(공식 데이터 사전 참고) |
| 시점·종점 노드 | `startNodeId`, `endNodeId` | 차량 진행 기준 시작·끝 노드 ID |
| 도로등급 | `도로등급`(CSV)·응답 필드 | 아래 `101`–`108` 코드 |
| 도로번호·명칭 | `roadName`, 노선 마스터 | |
| 연장 | 길이(m) | |
| 관리기관 | 도로관리기관 | |

상세 코드표는 `backend/data/its/road_rank_codes.json` 과 동일하다.

## 도로등급 코드 (`101`–`108`)

| code | 의미 |
|------|------|
| 101 | 고속도로 |
| 102 | 도시고속화도로 |
| 103 | 일반국도 |
| 104 | 특별·광역시도 |
| 105 | 국가지원지방도 |
| 106 | 지방도 |
| 107 | 시군도 |
| 108 | 기타 (시군도·기타는 도로번호 없을 수 있음) |

## 노드별 링크정보 CSV (대용량 마스터)

전국 링크 지오메트리·속성용. **레포에는 원본을 두지 않는다.** 로컬 예: `노드별 링크정보_2023 (1).csv`.

| 열 | 설명 |
|----|------|
| 링크아이디 | 링크 ID (`LINK_ID`) |
| 시점노드 / 시점노드명 | |
| 종점노드 / 종점노드명 | |
| 도로등급 | 위 101–108 |
| 도로명 | |
| 연장(M) | |
| 링크권역 | 예: 시·도 단위 권역명 |

- **인코딩**: `CP949`(EUC-KR 계열). 파이썬은 `encoding="cp949"` 로 열 것.
- 첫 데이터 행 다음에 영문 괄호 설명 행이 붙은 파일이 있을 수 있어, 파싱 시 `(` 로 시작하는 행은 건너뛴다.

## 본선 구간 정보 (별도 엑셀)

고속·국도 **본선 구간** 단위 ID(`MLN_SCTN_ID`)·방향·노선번호는 사용자 제공 「본선 구간 정보」 시트를 참고한다. 예측 API `sectionId` 후보로 쓸 수 있다. (레포에는 요약만 두고, 원본은 로컬 보관 권장.)

## 관련 코드 모듈

- `frontend/src/lib/its-route-numbers.json`, `its-road-rank-codes.json` — 위 모듈이 import
- `frontend/src/lib/its-map-tiles.ts` — 실시간 소통 **WMTS** 타일 URL
