# Cursor vs Claude Code — 역할 분담 가이드

델파이 레거시 포팅 프로젝트에서 **Cursor**를 메인으로 두고 **Claude Code**를 병행할 때, 작업 성격에 따라 도구를 나누는 기준을 정리합니다.

## 한 줄 요약

| 도구 | 한 줄 |
|------|--------|
| **Cursor** | 에디터 안에서의 대화형 개발, 국소 수정, 리뷰, 규칙 기반 일관성 |
| **Claude Code** | 터미널·에이전트 기반의 넓은 범위 작업, 반복·스크립트·일괄 탐색 |

---

## Cursor에 맡기기 좋은 작업

- **파일/함수 단위** 수정: 특정 `.pas` / `.dpr` / 웹 쪽 `frontend`·`backend`의 국소 변경
- **컨텍스트가 에디터에 열려 있는 상태**에서의 질문·리팩터·이름 변경
- **프로젝트 규칙 반영**: `.cursor/rules`, `AGENTS.md` 등과 맞춘 코딩 스타일·금지 사항 준수
- **대시보드·JSON·문서** 소규모 동기화 (필드 추가, 문구 수정, 섹션 정리)
- **리뷰·설명**: “이 호출 흐름이 뭐지?”, “여기 포팅 시 리스크는?” 같은 읽기 중심 Q&A
- **Git diff와 함께** 보이는 변경의 마무리·정리

**델파이 관점**: 폼 이벤트, 특정 프로시저, uses 절 정리 등 **한 화면/한 유닛 안**에서 돌아가는 작업에 적합합니다.

---

## Claude Code에 맡기기 좋은 작업

- **디렉터리·다수 파일**을 건너는 탐색: “이 패턴이 레포 어디에 있나”, “이 컴포넌트명 전부 나열”
- **반복 패턴 일괄 적용**: 이름 규칙, 주석 블록, 스텁 파일 생성 (단, **컴파일 검증은 별도**)
- **스크립트 작성·실행**: Python 도구 연동, 작은 CLI, 리포트 생성, `tools/` 보조 자동화
- **문서·체크리스트 대량 초안**: 마이그레이션 노트, 인벤토리 표, 이슈 목록 템플릿
- **터미널과 붙는 작업**: `git` 상태 정리, 브랜치별 작업, 로그 기반 분석 (정책 허용 범위 내)
- Cursor가 열어두지 않은 경로까지 **한 번에 맥락을 잡아야 할 때**

**델파이 관점**: 유닛 수가 많을 때 **교차 참조·목록화·1차 분류**에는 유리합니다. 다만 **문법·런타임 정확성**은 Delphi 컴파일/실행으로 확인하는 전제를 둡니다.

---

## 겹치기 쉬운 구간 (주의)

다음은 **둘 다 잘하지만**, 동시에 두 도구에 맡기면 **충돌·중복**이 나기 쉽습니다.

- 같은 파일의 연속 수정
- 대규모 리네임 / 구조 변경
- `dashboard/data/*.json` 전체 일괄 변경

**권장**: 한 작업 단위(브랜치 또는 디렉터리 또는 “오늘 할 일 한 장”)는 **한 도구만 쓰기**로 끊거나, Claude Code는 **분석·패치 초안만** 하고 적용은 Cursor에서 하기.

---

## 의사결정 체크리스트

1. 지금 손대는 범위가 **한 파일 또는 소수 파일**인가? → **Cursor** 우선
2. **10개 이상 파일** 또는 **전 레포 패턴**이 필요한가? → **Claude Code** 후보
3. **프로젝트 규칙·린트·즉시 미리보기**가 중요한가? → **Cursor** 우선
4. **스크립트·명령 실행·배치**가 핵심인가? → **Claude Code** 후보
5. 델파이 **컴파일 결과가 곧 진실**인 변경인가? → 어느 도구든 **작은 커밋 + 빌드**로 검증

---

## 델파이 소스 전용 메모

- LLM은 Pascal/Delphi에 대한 **훈련 밀도가 상대적으로 낮을 수 있음** → 프롬프트에 **시그니처·uses·에러 메시지**를 짧게라도 포함
- **구문 트리·정적 분석**은 기존 `tools/` 파이프라인을 우선하고, LLM은 **의미 정리·포팅 후보·문서화** 보조로 두면 안전
- 기업/고객 **코드 반출 정책**에 맞는 API(엔터프라이즈/온프레 등)를 먼저 확정한 뒤 도구 선택

## 문서 유지

역할 분담이 바뀌면 이 파일의 표와 체크리스트만 고쳐도 됩니다. 팀 합의된 **“Claude Code 전용 브랜치”** 같은 규칙이 생기면 한 절을 추가하는 것을 권장합니다.
