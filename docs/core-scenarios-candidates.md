# 핵심 시나리오 후보 (5~10개 합의용)

| 항목 | 내용 |
|------|------|
| **목적** | 웹 베타 오픈 시 **반드시 돌아가야 하는 흐름**을 5~10개로 압축·합의 |
| **담당** | 기획자 + 메인개발자 |
| **합의 기한** | **2026-07-10** (이 날짜까지 표 ✅·§3 상세·§4 산출물명 초안 확정) |

[`docs/delphi-developer-confirmation-checklist.md`](delphi-developer-confirmation-checklist.md) §6.1·§6.2와 함께 쓰며, 합의 후 `migration/contracts/`·`migration/test-cases/`의 입력으로 직접 연결됩니다.

> 1차 후보(C1~C10)는 **레거시 정적 분석(L1·L2)** 근거입니다. **확장 후보(C11~)**는 역할·예외·동등성·운영(병행) 축을 넣어 **같은 업무를 “다양한 각도”로 검증**하기 위한 추가 안입니다. 회의에서 확정된 항목에 ✅ 표시 후, 미선택은 백로그로 보냅니다.

## 0. 다양성 축 (후보를 고를 때 체크)

한 축에만 몰리지 않도록, **최종 5~10개**에 아래를 가능하면 **골고루** 반영합니다.

| 축 | 질문 | 예시에 대응하는 후보 |
|----|------|----------------------|
| **역할** | 영업·창고·관리(회계) 중 누가 쓰나? | C2·C6·C5·C10 등 |
| **DB 리스크** | 읽기만 / 쓰기 / 삭제·마감 | DEC-003 순서와 맞춰 조합 |
| **통합** | 인쇄·바코드·외부 장비 | C7·C8, DEC-004 |
| **예외·복구** | 취소, 마감 후 차단, 세션 끊김 | C11·C12·C13 |
| **동등성** | 레거시와 숫자·집계가 같아야 하나? | C14, 체크리스트 §6.2 |
| **운영** | 델파이 병행·동시 편집 | C15, 체크리스트 §5.3 |

## 1b. 확장 후보 (다양성·엣지 — 10개 안에 넣을지 선택)

아래는 **C1~C10과 중복되지 않는 “검증 관점”**입니다. 최종 10개를 채울 때 **일부를 C1~C10의 하위 시나리오(§3)**로 흡수해도 되고, 별도 ID로 잡아도 됩니다.

| # | 흐름 | 검증 포인트 | DB/비고 | 결정 |
|---|------|-------------|---------|------|
| C11 | **출고·주문 수정/취소(마감 전)** | 저장 후 수정·삭제, 재집계와 일관성 | UPDATE/DELETE, 트랜잭션 경계 | ☐ |
| C12 | **마감 후 편집 차단·오류 메시지** | “마감됨” 분기가 레거시와 동일한가 | 규칙 전용, DB는 상황별 | ☐ |
| C13 | **세션 만료·재로그인·권한 부족** | 토큰 만료 후 재진입, f권한 거부 시 UI | C1·C10과 연계 E2E | ☐ |
| C14 | **핵심 보고서·집계 동등성** | §6.2 목록 중 1~2개, 숫자 대조 | SELECT/뷰, 체크리스트 6.2 | ☐ |
| C15 | **델파이 병행 시 충돌 방지(잠금/알림)** | 동일 건 동시 편집 시 동작 | 정책·OQ에 의존, §5.3 | ☐ |

**권장**: C11·C12는 출고·정산 품질에 직결되므로 **C2/C5 채택 시 하위 시나리오로 필수**에 가깝습니다. C14는 §6.2 목록이 정해지면 ID를 구체 보고서명으로 바꿉니다.

## 2. 우선순위 가이드

[`legacy-analysis/decisions.md`](../legacy-analysis/decisions.md) **DEC-003 권장 포팅 순서**(읽기전용 → 신규등록 → 수정/취소 → 배치/인쇄/장비 → 고객사 커스터마이징)에 맞춰 다음 묶음을 권장합니다.

- **베타 필수**(가장 적은 수): C1 + C2 + C7 (로그인·출고·인쇄)
- **베타 권장**(5개): + C6 + C8 (조회·바코드)
- **내부오픈 확장**(10개): + C3, C4, C5, C9, C10

**다양성을 넣은 8개 예시**(회의용 스타터 — 확정 아님):  
C1, C2+C11(출고 정상+수정/취소), C3, C5+C12(마감+마감후 차단), C6, C7, C8, C14(집계 1건 동등성).  
→ 역할·읽기/쓰기·인쇄·장비·예외·동등성을 한 번에 묶은 구성입니다. 남는 슬롯에 C4·C9·C10·C13·C15를 우선순위로 끼워 넣습니다.
