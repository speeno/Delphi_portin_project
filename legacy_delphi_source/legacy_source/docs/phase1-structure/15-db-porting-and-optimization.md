## 1. How data access works in this codebase

The Delphi app combines **in-memory `TClientDataSet`** (often via `Base10.OpenShow`), **string-built SQL** (`Sqlen`, `Sql.Add`), and legacy **middleware-style** calls such as `Base10.Seek_Name` / socket `RunSQL` (see [`Base01.pas`](../../Base01.pas)). A modern port typically replaces string SQL with **parameterized queries**, a single **connection pool**, and explicit **repositories** per aggregate.

## 4. Optimization and modernization playbook

1. **Soft delete / visibility**: `Base01` uses `D_Open` / `D_Select` with `` `Check` `` — ensure every query in the new stack applies the same rule or replace with explicit `deleted_at`.
2. **Hot keys**: Many tables filter on `Hcode`, `Gcode`, dates — add or verify **composite indexes** matching `WHERE` clauses from [`13-db-sql-references.csv`](13-db-sql-references.csv) snippets.
3. **Dead schema**: `Live only` tables — trace **MySQL user grants** and **application logs**; candidates for read-only archive or separate schema after sign-off.
4. **Missing tables** (`Code only`): run **integration tests** against live DB; fix synonyms or split multi-tenant DBs.
5. **Replace `Seek_Name` / socket SQL** with a typed API + transactions to simplify pooling and error handling.
6. **Security**: move from shared `root` to **least-privilege** users per service; rotate passwords exposed outside secret stores.
