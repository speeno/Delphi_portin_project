# 알려진 위험 (Known Risks)

## 형식
- **ID**: RISK-XXX
- **위험**: 구체적 위험 내용
- **확률**: 높음 / 중간 / 낮음
- **영향**: 높음 / 중간 / 낮음
- **대응 전략**: 위험을 줄이기 위한 행동
- **담당**: 누가 대응하는가
- **상태**: 감시중 / 발생함 / 해소됨

---

## 위험 목록

### RISK-001: 레거시 SQL 동작 차이
- **위험**: 델파이에서 실행되는 SQL의 동작이 웹 API에서 재현 시 미묘하게 다를 수 있음 (트랜잭션 격리 수준, 암묵적 타입 변환, NULL 처리 등)
- **확률**: 높음
- **영향**: 높음
- **대응 전략**: Capture Harness로 실제 쿼리를 캡처하고, Comparison Harness로 신/구 결과를 자동 비교
- **담당**: M + S-Mid
- **상태**: 감시중

### RISK-002: 고객사 커스터마이징 누락
- **위험**: 코드 내 고객사별 분기가 많아서 포팅 시 일부가 누락될 수 있음
- **확률**: 높음
- **영향**: 중간
- **대응 전략**: Customer Variant Matrix 작성 + 가드레일에 "고객사 분기 삭제 금지" 명시 + Capture 데이터로 실사용 분기 식별
- **담당**: P + M
- **상태**: 감시중

### RISK-003: 장비 브리지 호환성
- **위험**: 현장 프린터/스캐너가 브라우저 환경에서 동작하지 않을 수 있음
- **확률**: 중간
- **영향**: 높음
- **대응 전략**: Sprint 3에서 조기 PoC 실시 + Capture Harness에 장비 명령 캡처 포함
- **담당**: S-Mid
- **상태**: 감시중

### RISK-004: 성능 저하
- **위험**: 웹 전환 후 응답 시간이 델파이 Desktop 대비 느려질 수 있음
- **확률**: 중간
- **영향**: 높음
- **대응 전략**: Comparison Harness에서 응답시간 비교 + 프로파일링 + 캐싱 전략
- **담당**: M
- **상태**: 감시중

### RISK-005: 현업 저항/교육 부족
- **위험**: 기존 시스템에 익숙한 사용자가 웹 전환을 거부하거나 교육 부족으로 오사용
- **확률**: 중간
- **영향**: 중간
- **대응 전략**: Routing Harness로 점진적 전환 + 즉시 롤백 보장 + 조기 데모
- **담당**: P
- **상태**: 감시중

### RISK-006: 데이터 정합성 문제
- **위험**: 전환 중 신/구 시스템 간 데이터 불일치 발생 가능
- **확률**: 중간
- **영향**: 높음
- **대응 전략**: Comparison Harness 상시 비교 + 자동 알림
- **담당**: M + S-Mid
- **상태**: 감시중

### RISK-007: 하네스 자체 장애
- **위험**: 하네스 인프라(캡처/비교/라우팅) 자체에 장애가 발생하여 운영에 영향
- **확률**: 낮음
- **영향**: 중간
- **대응 전략**: 하네스 우회 경로(바이패스) 설계 + 하네스 운영 런북 작성
- **담당**: M + S-Mid
- **상태**: 감시중

### RISK-008: DFM→HTML 미리보기와 실제 UI 괴리
- **위험**: `.dfm`에서 생성한 HTML 미리보기를 최종 웹 UI로 오인하면 기대와 다른 배치·폰트·그리드 동작으로 이해관계자 혼선이 생길 수 있음
- **확률**: 중간
- **영향**: 중간
- **대응 전략**: 레이아웃 JSON·`layout_ir`를 기준으로 삼고, 한계·비범위는 [`docs/dfm-html-layout-limitations.md`](../docs/dfm-html-layout-limitations.md)에 명시; 시각 회귀는 선택·`workflow_dispatch`로 flaky 리스크를 통제
- **담당**: 포팅/분석 담당
- **상태**: 감시중
